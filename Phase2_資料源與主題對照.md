### 美股 / 韓股 / 日股（尚未接入）

一個殘酷現實：**Yahoo Finance 官方 API 已被弃用**，靠爬蟲的 wrapper（yfinance 等）
隨時可能失效，不建議當主力。以下是 2026 年仍可用的替代方案，依適合你的程度排序。

| API | 免費額度（須複查） | 涵蓋市場 | 適合度 | 備註 |
|---|---|---|---|---|
| **Alpha Vantage** | 約 25 次/日、5 次/分 | 美股佳，全球部分 | 美股首選 | 官方授權資料、格式乾淨、有 MCP 支援 |
| **Finnhub** | 約 60 次/分 | 美股佳，國際部分 | 美股次選 | 分鐘額度寬，適合逐檔查 |
| **Twelve Data** | 約 800 次/日 | 全球含日韓 | 日韓可考慮 | 日額度大，4 小時延遲（盤後夠用）|
| **FCS API** | 約 500 次/月 | 50+ 交易所含日本 | 備用 | 月額度小，測試用 |
| **iTick** | 基本報價宣稱不限次 | 美/日/韓/港/A股 | 亞股可考慮 | 較新、須複查穩定度與條款 |

### 給你的接入建議（重要）

1. **美股**用 Alpha Vantage 為主：它對美股涵蓋最完整、資料有官方授權、格式最乾淨。
   但免費額度極小（約 25 次/日），所以**不能逐檔抓**——策略是「只抓你主題對照表裡的成分股」，
   例如美股極細主題總共約 40-50 檔，配合每日 1 次更新，靠 caching 勉強在額度內。

2. **韓股 / 日股**用 Twelve Data 為主：它日額度大（約 800 次/日）、涵蓋日韓、盤後延遲可接受。
   同樣只抓對照表內的成分股。

3. **關鍵架構決定：不要即時、只做盤後一次更新。**
   四市場盤後時間不同（美股對台灣是隔天清晨、日韓接近台灣），
   GitHub Actions 分市場設不同 cron，各自在該市場收盤後跑一次，把額度用在刀口上。

4. **所有第三方 API 都要 API key。** 免費申請，但 key 不能寫進公開 repo（會被盜用）。
   正確做法：存進 GitHub 的 **Secrets**（Settings → Secrets and variables → Actions），
   在 workflow 裡用 `${{ secrets.ALPHAVANTAGE_KEY }}` 讀取。之後接美股時我再帶你設。

5. **匯率 / 指數**另需要來源，接入時再處理。
